# Software Architect

## Role
The Software Architect is responsible for the high-level design of the entire technical system. They make the big decisions about how different services interact, ensuring the system is maintainable, scalable, and resilient over years of growth.

## Responsibilities
- **Structural Design:** Choosing between monoliths vs. microservices vs. serverless.
- **Data Modeling:** Designing the overarching strategy for data flow and persistence across the company.
- **Tech Stack Selection:** Deciding which languages and frameworks will be used across teams.
- **Standard Setting:** Defining the "North Star" for code quality, documentation, and system design.

## Tools and Technologies
- **Diagramming:** Lucidchart, Miro, Mermaid.js, Archimate.
- **Architecture Patterns:** Domain Driven Design (DDD), Event-Sourcing, CQRS.
- **Cloud Design:** AWS/GCP Architecture Centers.
- **Infrastructure Strategy:** Kubernetes (high-level design), Kafka (for event-driven architectures).
